vsm Workflow¶

The basic stages in a vsm workflow are shown below.

Raw-processing Converting a textual corpus into a plain text file or a directory tree whose leaves are plain text files
Tokenization and Metadata Collection Words and various types of word contexts such as sentences, paragraphs, pages, articles, documents, abstracts or books are identified; as well, labels and other metadata are gathered and associated to contexts
Corpus Generation A tokenized plain-text corpus is encoded as an integer array; contexts are stored as indices into the corpus array and additional metadata are stored in records associated to the contexts; stoplists and low-frequency filters are also applied at this stage.
Model training A Corpus object is given as input to a mathematical model for representing a corpus’s semantics; provisions for running a parallel implementation of a model (e.g., initiating an IPython cluster) may be made.
Viewing and Analyzing Results A Corpus object and an associated model object are given to a viewer class so that the user may, for example, examine the distances between semantic representations of words, contexts (documents) or topics; graphical display and notebook creation options are provided by a IPython notebook server.
