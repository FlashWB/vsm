toy_corpus¶

vsm.corpus.util.corpusbuilders.toy_corpus(plain_corpus, is_filename=False, nltk_stop=False, stop_freq=0, add_stop=None, metadata=None)¶

toy_corpus is a convenience function for generating Corpus objects from a given string or a single file.

toy_corpus will perform both word and document-level tokenization. It will also strip punctuation and arabic numerals outside the range 1-29. All letters are made lowercase.

Document tokens are delimited by two or more line breaks. E.g.,

<document 0>

<document 1>

...

<document n>

where <document i> is any chunk of text to be tokenized by word.

Parameters:
  • plain_corpus (string-like) – String containing a plain-text corpus or a filename of a file containing one.
  • is_filename (boolean, optional) – If True then plain_corpus is treated like a filename. Otherwise, plain_corpus is presumed to contain the corpus. Default is False.
  • nltk_stop (boolean, optional) – If True then the corpus object is masked using the NLTK English stop words. Default is False.
  • stop_freq (int, optional) – The upper bound for a word to be masked on the basis of its collection frequency. Default is 0.
  • add_stop (array-like, optional) – A list of stop words. Default is None.
  • metadata (array-like, optional) – A list of strings providing metadata about the documents. If provided, must have length equal to the number of documents. Default is None.
Returns:

c : a Corpus object Contains the tokenized corpus built from the input plain-text corpus. Document tokens are named documents.
See Also:

vsm.corpus.Corpus, vsm.corpus.util.paragraph_tokenize(), vsm.corpus.util.apply_stoplist()
